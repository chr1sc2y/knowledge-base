# Providence Knowledge Base

A calm, bilingual knowledge base for clear, durable explanations.  
Deployed at https://knowledge.prov1dence.top/

## Why "Not Secure"?

The site uses a custom domain on GitHub Pages. HTTPS requires proper configuration:

1. In the GitHub repo, go to **Settings → Pages**.
2. Under "Custom domain", enter `knowledge.prov1dence.top` (or confirm it is already set from the `CNAME` file).
3. Click **Save**.
4. Wait for the DNS check to pass and the certificate status to show "Issued" (may take a few minutes to ~1 hour; GitHub provisions via Let's Encrypt).
5. Check **Enforce HTTPS**.

If the certificate still shows a mismatch (serving `*.github.io` instead of your domain):
- Toggle the custom domain off and back on.
- Ensure your DNS CNAME for the subdomain points to `chr1sc2y.github.io.` (no proxy / "DNS only" if using Cloudflare during initial setup).
- The `CNAME` file in the repo root must be present and the Pages source must be "GitHub Actions".

Once enforced, the site redirects HTTP → HTTPS and browsers stop warning.

## Usage (Local)

```bash
python scripts/build.py
open dist/index.html
```

## Adding Content

1. Place Chinese articles in `articles/cn/`.
2. Place matching English articles in `articles/en/`.
3. Register both paths and localized metadata in `data/pages.json`.
4. Add external references in `data/links.json` if needed, but keep references off the home page.
5. Push to `master`. GitHub Actions builds `dist/` and deploys to GitHub Pages.

## Design Notes

- Editorial template: warm paper background, serif display headings, restrained accent color, and section-by-section reading flow.
- Every public page should have Simplified Chinese and English versions with a matching language switch.
- Home page is a quiet index, not a card grid and not a references wall.
- Articles explain topics progressively from basic concepts and mechanisms to practical implications. Avoid labeling depth of explanation as literal school stages unless explicitly requested.
- Use cards sparingly. Prefer typography, rules, tables, notes, and a small number of framed interactive tools.

## Tech

- Static site generated by `scripts/build.py`.
- Deployed via `.github/workflows/pages.yml` using official `upload-pages-artifact` + `deploy-pages`.
- Custom domain defined by root `CNAME` file.

## Contributing

Keep additions focused and high-signal. Maintain Chinese and English together. Prefer precise, citable explanations over long prose. Update the build before pushing.
