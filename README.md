# Providence Knowledge Base

A clean, private knowledge base for high-value learning pages and curated references.  
Deployed at https://knowledge.prov1dence.top/

## Why "Not Secure"?

The site uses a custom domain on GitHub Pages. HTTPS requires proper configuration:

1. In the GitHub repo, go to **Settings → Pages**.
2. Under "Custom domain", enter `knowledge.prov1dence.top` (or confirm it is already set from the `CNAME` file).
3. Click **Save**.
4. Wait for the DNS check to pass and the certificate status to show "Issued" (may take a few minutes to ~1 hour; GitHub provisions via Let's Encrypt).
5. Check **Enforce HTTPS**.

If the certificate still shows a mismatch (serving `*.github.io` instead of your domain):
- Toggle the custom domain off and back on.
- Ensure your DNS CNAME for the subdomain points to `chr1sc2y.github.io.` (no proxy / "DNS only" if using Cloudflare during initial setup).
- The `CNAME` file in the repo root must be present and the Pages source must be "GitHub Actions".

Once enforced, the site redirects HTTP → HTTPS and browsers stop warning.

## Usage (Local)

```bash
python scripts/build.py
open dist/index.html
```

## Adding Content

1. Place a standalone HTML page in `pages/`.
2. Register it in `data/pages.json` with title, category, tags, and path.
3. Add external references in `data/links.json` if needed.
4. Push to `master`. GitHub Actions builds `dist/` and deploys to GitHub Pages.

## Design Notes

- Apple-inspired: generous whitespace, crisp system typography, minimal chrome, subtle interactions, high scannability.
- All content is in English.
- Keep pages concise. Focus on depth and clarity over volume. Avoid redundant explanations or decorative UI.
- Professional design tools worth using before coding:
  - **Figma** (industry standard for UI/UX, great for knowledge-base layouts, prototypes, and design systems).
  - Apple Human Interface Guidelines (HIG) and SF Symbols / typography references.
  - Optional: Framer for high-fidelity interactive mocks, or Penpot (open source).

## Tech

- Static site generated by `scripts/build.py`.
- Deployed via `.github/workflows/pages.yml` using official `upload-pages-artifact` + `deploy-pages`.
- Custom domain defined by root `CNAME` file.
