Identity-based Policy vs. Resource-based Policy

IAM Permission Boundary

A permission boundary limits the maximum permissions an IAM user/role can obtain.

STS (AWS Security Token Service)

Creates temporary security credentials.

Trust Policy & Permission Policy

A trust policy specifies which principal can perform which action.

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "AWS": "arn:aws:iam::123456789012:user/Alice"
      },
      "Action": "sts:AssumeRole"
    }
  ]
}

A permission policy defines which AWS resources and actions the user or service holding the role can access.

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "s3:ListBucket",
      "Resource": "arn:aws:s3:::example_bucket"
    },
    {
      "Effect": "Allow",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::example_bucket/*"
    }
  ]
}

Cross-Region Database

RDS

  1. Cross-Region Read Replicas (read only): a read replica can be promoted to a standalone primary instance for disaster recovery when the primary region fails
  2. Cross-Region Backups and Restore: restore the database from backups when the primary region fails

Aurora

  1. Global Database: RTO < 1 min
  2. Cross-region DB cluster: supports cross-region read/write

DynamoDB

  1. Global Tables: replicas in other regions stay available when one region fails
  2. Cross-Region Backup and Restore: restore the table from backups when the primary region fails

On-Demand

  1. EC2 On-Demand Instances
  2. RDS On-Demand Instances
  3. DynamoDB On-Demand Capacity Mode

Edge-optimized

  1. API Gateway Edge-Optimized Endpoints: route API requests to the nearest CloudFront edge location
  2. S3 Transfer Acceleration: use CloudFront edge locations to accelerate transfers
  3. Lambda@Edge: run Lambda functions at CloudFront edge locations

Saving Plan

  1. EC2 Instance Savings Plans: target specific EC2 instance types, higher discount, similar to Reserved Instances
  2. Compute Savings Plans: apply to all AWS compute, including EC2, Lambda, Fargate

KMS vs. Secrets Manager

  • KMS: manage encryption keys
  • Secrets Manager: manage credentials such as SSH key pairs

Authentication

SSO (Single Sign-On)

  • an authentication mechanism
  • log in to multiple systems with one set of credentials
  • access is granted automatically without re-entering credentials

IdP (Identity Provider)

  • SSO is usually implemented through an IdP
  • responsible for verifying user identity
  • implemented via protocols such as OIDC, OAuth, SAML

Route 53

DNS resolution maps domain names to AWS resources (e.g. EC2, S3) or external IP addresses, supporting these record types:

  • A record: maps a domain to an IPv4 address
  • AAAA record: maps a domain to an IPv6 address
  • CNAME record: maps one domain to another domain
  • MX record: defines mail servers
  • TXT record: stores arbitrary text, commonly for domain verification

HTTP API vs. REST API

  • HTTP API: simple microservice architectures, basic integration and authentication
  • REST API: more comprehensive, integrates directly with DDB

NACL vs. Security Group

  • NACL: stateless, each direction (inbound & outbound) configured independently; supports Allow and Deny
  • Security Group: stateful, an allowed inbound request automatically allows the returning outbound response; Allow only

Connectivity (not over the internet)

  1. Direct Connect (on-premises ↔ VPC)
  2. Site-to-Site VPN (on-premises ↔ VPC)
  3. PrivateLink (VPC ↔ services)
  4. VPC Peering (VPC ↔ VPC)

VMware Migration

Migrating from VMware infrastructure to Amazon EC2:

  1. Use the VMware vSphere client to export the application as an image in Open Virtualization Format (OVF)
  2. Create an S3 bucket in the destination AWS region to store the image
  3. Create and apply an IAM role for VM Import
  4. Use the AWS CLI to run the EC2 import command

Others

IP Addresses

  • Class A: 8-bit prefix, e.g. 44.x.x.x
  • Class B: 16-bit prefix, e.g. 44.66.x.x
  • Class C: 24-bit prefix, e.g. 192.168.31.x
  • CIDR: variable prefix located via a subnet mask (VLSM), e.g. 192.0.2.0/24 (24-bit prefix)
  • Network prefixes: IP address ranges associated with an on-premises network

SSL

  • Amazon-issued public certificates cannot be installed on an EC2 instance
  • To enable end-to-end encryption, you must use a third-party SSL certificate

Application Auto Scaling

Automatically increases or decreases resource capacity based on demand, for EC2, ECS, Lambda, DynamoDB, and Aurora.

Common Exam Mistakes

  • Reserved Instances (RI) suit long-term reservation of EC2 instances but do not apply to other compute types (such as Fargate and Lambda) and can be inflexible under fluctuating demand.
  • Compute Savings Plans offer flexible savings across all compute types (EC2, Fargate, Lambda), can be applied across accounts and regions, and provide the most flexibility and savings — the best fit for varied accounts and fluctuating demand within an organization.
  • To handle cross-origin issues for API Gateway, enable CORS directly on API Gateway.
  • Configure the Auto Scaling group with an AMI that has the CodeDeploy agent pre-installed to simplify deployment management.
  • Provisioned IOPS don't scale on demand.
  • Converting API Gateway to an edge-optimized endpoint and enabling caching is cheaper than ElastiCache.
  • When using EC2 instances as network proxies (NAT), disable the source/destination check.
